# /dashboard — гид по проекту finproekt_dashbord

Ты работаешь с проектом **GetCourse Dashboard Creator** — настольное Python-приложение,
которое автоматически создаёт аналитический дашборд в GetCourse через Playwright.

---

## Структура файлов

```
finproekt_dashbord/
├── main.py       — Tkinter GUI: поля ввода, кнопки, лог-виджет
├── creator.py    — вся автоматизация Playwright: логин, дашборд, виджеты
├── config.py     — загрузка/сохранение settings.json
├── settings.json — сохранённые credentials (gc_url, gc_email, gc_password, tg_bot)
├── requirements.txt — единственная зависимость: playwright>=1.40
├── README.md     — инструкция для пользователя
├── screenshots/  — скриншоты каждого шага (создаются при запуске)
└── debug/        — дампы элементов формы (только режим диагностики)
```

**Ключевые функции в creator.py:**

| Функция | Что делает |
|---------|------------|
| `login` | Логин в GetCourse |
| `go_to_new_dashboard` | Переход и клик «Создать дашборд» |
| `set_dashboard_name` | Ввод названия, клик «Создать», ожидание страницы |
| `open_add_widget` | Клик «Добавить блок», ожидание `input.title-input` |
| `set_widget_title` | Заполнение названия виджета |
| `select_source` | Выбор источника данных (`select.context-selector`) |
| `select_date_field` | Выбор поля даты |
| `set_group_filter` | Фильтр «В группе» через кастомный Select2 |
| `set_offer_filter` | Фильтр «Предложение» через Select2 |
| `save_widget` | Клик «Сохранить» в модале, ожидание закрытия |
| `save_dashboard` | Клик «Сохранить дашборд» |
| `create_dashboard` | Главный оркестратор: логин → дашборд → 3 виджета → сохранение |
| `diagnostic_run` | Диагностический режим: разведка формы, сохранение в debug/ |

## Как запустить

```bash
# Установка (один раз)
pip install -r requirements.txt
playwright install chromium

# Запуск
python main.py
```

GUI открывается — пользователь заполняет поля и жмёт «Создать дашборд».  
Браузер открывается автоматически и **остаётся открытым** после завершения для проверки.

## Три виджета (текущий scope)

```python
steps = [

    ("Посещения всего",    widget_visits(page, tag, log_fn)),
    ("Платное участие",    widget_paid(page, tag, offer_name, log_fn)),
```

В `creator.py` есть ещё 8 функций виджетов (bot, banner, pokoi, zerocoder, buttons, speaker, orders, max) — они написаны, но **не подключены** в `create_dashboard`. Не трогай их без явного запроса.

## Известные особенности GetCourse

**Select2 для «В группе»** — кастомный, нестандартный:
- Кликать по `.rules-container .select2-container` целиком (не по `a.select2-choice`)
- `wait_for(state="visible")` на `a.select2-choice` таймаутит — не использовать
- Реализовано в `_open_last_select2` и `_select2_pick`

**select_source (выбор типа объекта):**
- После сохранения виджета форма «помнит» предыдущий выбор
- Нужен `dispatch_event('change')` после `select_option` — иначе форма не перерисовывается
- Без этого второй виджет не получает раздел фильтров

**Тайминги между виджетами:**
- После `save_widget` — sleep 1.5с (не меньше, иначе стale-элементы в DOM)
- После `select_source` — sleep 2.0с (ждём перерисовки формы)

## Как вносить изменения

### Добавить новый виджет в create_dashboard
1. Убедись, что функция `widget_*` уже есть в creator.py (скорее всего есть)
2. Добавь её в список `steps` в `create_dashboard`
3. Проверь: нужен ли ей новый параметр (tg_bot, speaker_utm и т.д.)

### Изменить поведение существующего виджета
- Все параметры виджетов (название, имя группы) формируются через `tag`
- Шаблон имён групп: `[{tag}] Название группы`

### Добавить новое поле в GUI
- Поле добавляется в `_build_ui` в `main.py`
- Значение передаётся в `creator.run(...)` через новый параметр
- Не забудь добавить его в `_validate_webinar` если поле обязательное

## Проверка ошибок

**Смотреть в первую очередь:**
1. Лог в GUI — текст ошибки выводится там
2. `screenshots/` — скриншоты каждого шага, видно где остановилось
3. `screenshots/w2_after_source.png` — специальный скриншот после выбора источника у 2-го виджета

**Частые ошибки:**

| Симптом | Вероятная причина | Где смотреть |
|---------|------------------|--------------|
| Второй виджет не выбрал тип | `change` event не сработал | `select_source` в creator.py |
| «Добавить условие» не найдено | Форма не перерисовалась после select_source | sleep после select_source |
| Таймаут на Select2 | Кликаем по `a.select2-choice` вместо контейнера | `_open_last_select2` |
| Виджет создан без фильтра | Имя группы не совпадает с GetCourse | Проверить точное написание тега |
| Ошибка логина | Неверный URL/email/пароль | settings.json |

**Запустить диагностику** (разведка формы):
- В GUI кнопка «Разведка формы»
- Создаёт дашборд `ДИАГНОСТИКА_УДАЛИТЬ`, сохраняет debug/*.txt и debug/*.png
- После диагностики удалить этот дашборд вручную в GetCourse

## Подготовка к публикации

Перед публикацией в репозиторий:

1. **Убрать credentials** — `settings.json` не должен попасть в git (добавь в `.gitignore`)
2. **Очистить скриншоты** — папки `screenshots/` и `debug/` тоже в `.gitignore`
3. **Проверить README** — актуален ли список виджетов, корректны ли примеры
4. **Проверить requirements.txt** — `pip freeze` для уточнения версий
5. **Добавить `.gitignore`** если его нет:

```
settings.json
screenshots/
debug/
__pycache__/
*.pyc
```

6. **Проверить что скрипт запускается** на чистой среде: `pip install -r requirements.txt && playwright install chromium && python main.py`
